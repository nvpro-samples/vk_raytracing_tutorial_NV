This is an extension of the Vulkan ray tracing [tutorial](vkrt_tutorial.md.html). (insert setup.md.html here) ## High-Level Implementation From a high-level view, we will * Add two million axis-aligned bounding boxes in a BLAS * Add two materials, one representing a sphere and the other representing a cube. * Objects will alternate between these two materials. To do this, we will need to: * Add an intersection shader (.rint) * Add a new closest hit shader (.chit) * Create `VkGeometryNV` objects from `VkGeometryAABBNV` objects ## Creating all spheres In the `HelloVulkan` class in `hello_vulkan.h`, we will add the structures we will need. First, we need to add the structure that defines a sphere: ~~~~ C++ struct Sphere { nvmath::vec3f center; float radius; }; ~~~~ Then we need the `Aabb` structure that will be used to hold all the spheres and also to create the BLAS (using `VK_GEOMETRY_TYPE_AABBS_NV`). ~~~~ C++ struct Aabb { nvmath::vec3f minimum; nvmath::vec3f maximum; }; ~~~~ All the information will need to be held in buffers, which will be available to the shaders.
